Interesting Idea

Bob is teaching Alice about proper code and proper documentation. Alice, being familiar with competitive programming, is unfamiliar with these concepts.

He asks for your help with an interesting example.

Task

Write an interesting program in one of the given languages, that accepts input through stdin and gives output through stdout.

Furthermore, make sure that your program is well-documented and that it is as easy as possible for someone to read your source code, even if they do not know the language.

You will receive 10 points for submitting this problem (and having it compile properly if needed), but at the end of the contest, only your latest submission will be judged manually.
You will be marked out of 50 points for proper, complete, and precise documentation and code style.
You will be marked out of 50 points based on how interesting or impressive your program is. Remember that it can only be interesting and impressive if the judges can understand it.
